# Problématique

## Questions auxquelles le projet répond

### Comment éviter le vibe coding avec Claude Code ?

Le vibe coding consiste à improviser prompt par prompt avec une IA de code. Ça marche pour des petits trucs, mais sur un projet structuré le résultat part dans toutes les directions. Le SDD propose une alternative : préparer le contexte avant de lancer le code.

### Une IA peut-elle respecter des règles métier précises ?

Le calculateur d'escaliers doit respecter les règles du Code de construction du Québec. Si Claude Code respecte ces règles à partir de la documentation préparée, c'est une démonstration concrète que le SDD fonctionne pour des projets sérieux.

### Comment préparer un contexte parfait pour l'IA ?

Le projet teste une combinaison concrète : un `CLAUDE.md` à la racine, un wiki Obsidian dans `docs/`, et des specs SpecKit dans `specs/`.

### Le SDD permet-il une vraie autonomie de Claude Code ?

L'objectif est que Claude Code puisse implémenter le MVP avec un minimum d'intervention manuelle, en se référant à la documentation préparée plutôt qu'en posant constamment des questions.
